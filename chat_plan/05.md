[Task 5] 채팅 UX / 스크롤 엣지 케이스 튜닝

마지막으로, 채팅 UX를 프로덕션 수준으로 다듬자.

요구사항:

1) 스크롤 UX
   - 이전 메시지 로딩 시:
     - startReached → onLoadMoreBefore → 새로운 페이지 prepend 될 때
       스크롤 위치가 튀지 않도록 Virtuoso의 firstItemIndex / totalCount 를 어떻게 썼는지 정리해라.:contentReference[oaicite:11]{index=11}
   - 새 메시지 유입 시:
     - 사용자가 아래에 있을 때만 followOutput 또는 수동 scrollToIndex 를 써서
       자동 스크롤하도록 로직을 정리해라.:contentReference[oaicite:12]{index=12}
     - 아래에 있지 않을 때는 "새 메시지 N개" 배지를 페이지 하단/헤더에 노출하는 패턴을 추가해라.:contentReference[oaicite:13]{index=13}

2) 읽지 않음 경계
   - lastReadAt 또는 lastReadMessageId 기준으로
     - "여기까지 읽음" 구분선을 ChatMessageList 안에서 렌더하는 방식을 설계해라.
   - 이 경계가 스크롤/가상화에 의해 사라졌다가 다시 나타날 때
     안정적으로 동작하는지 확인할 수 있는 테스트 아이디어를 적어라.

3) 접근성
   - 메시지 리스트 컨테이너에 적절한 role/aria 속성을 제안해라
     (예: role="log", aria-live="polite" 등).:contentReference[oaicite:14]{index=14}
   - 키보드로 "맨 아래로 이동" / "새 메시지 위치로 이동"을 할 수 있는
     최소 단축키/포커스 이동 설계를 제안해라.

4) 출력 형식
   - 수정된 코드(구분선/배지/aria 관련 부분)
   - UX 패턴 체크리스트 (bullet 형식)
   - 남아 있는 리스크/테스트 할 만한 시나리오 목록
   을 포함해라.
